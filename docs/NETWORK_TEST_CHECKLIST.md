# 联网版测试清单

以下清单用于发布前验证。快速的本机联网流程可先执行：

```bash
ADMIN_PASSWORD='管理员密码' node tools/network-smoke.mjs
```

该脚本会验证认证、角色权限、聊天、双方就绪、开局、结束重置和准备阶段编辑，不替代
下面的 UI、高 DPI 和异常网络人工检查。

## Docker 与账号平台

- [ ] 从空数据卷执行 `docker compose -f deploy/compose.yml up -d --build`，两个健康检查均通过。
- [ ] `http://localhost:8080` 可访问，错误管理员密码被拒绝，正确密码可进入控制台。
- [ ] 创建导演、编辑、红方、蓝方账号；用户名重复、短密码和非法角色被拒绝。
- [ ] 修改账号显示名、角色、密码和启用状态后，列表与统计数字同步更新。
- [ ] 停用、改密、改角色或删除账号后，该账号已有联网会话失效。
- [ ] 管理员在网页修改密码后现有管理会话失效，新密码可重新登录。
- [ ] `./deploy/reset-admin.sh` 可交互重置管理员密码，且不影响兵棋账号。
- [ ] 容器重启后账号、管理员密码和联网场景仍从持久卷恢复。
- [ ] 推演运行中重启 `game-server` 后，时间、HP、FSM、航点、攻击目标和双方就绪/阶段从检查点恢复。
- [ ] 重复提交同一用户的同一 `commandId`（包括断线重连后）只执行一次，并返回相同结果。
- [ ] 宿主机除 `127.0.0.1:8080` 与 `127.0.0.1:8090` 外无额外服务端口暴露。

## 客户端模式与登录

- [ ] 客户端首次启动显示本地/联网模式选择，未选择时不能关闭对话框。
- [ ] 本地模式保留原有四视角、场景编辑和推演操作。
- [ ] 联网登录的服务器地址、用户名、密码错误时显示中文错误且不会进入推演页面。
- [ ] 四类账号登录后分别固定为导演、编辑、红方、蓝方页面，不能绕过 UI 切换角色。
- [ ] 设置面板能从本地切到联网，也能从联网切回本地。
- [ ] WebSocket 短暂中断时显示重连状态，服务恢复后自动重新认证并同步快照。
- [ ] 人为丢弃一个 delta 后，客户端停止应用后续增量并通过完整快照恢复一致状态。

## 准备阶段与权限

- [ ] 编辑席可增删改红蓝双方初始单位；推演开始后编辑界面变为只读。
- [ ] 红方只能编辑红方阵容，蓝方只能编辑蓝方阵容，篡改协议也不能操作对方单位。
- [ ] 任意场景变更都会清除红蓝双方就绪状态。
- [ ] 缺失、重复或已摧毁任一方指挥所时不能提交有效开局条件。
- [ ] 红蓝双方可独立提交和取消就绪，所有客户端实时显示双方状态。
- [ ] 只有双方均就绪且场景有效时，导演“开启推演”按钮可用。
- [ ] 非导演账号直接发送开局、暂停、调速、单步和结束消息均被服务器拒绝。

## 推演同步与视野

- [ ] 多客户端观察到的推演时间、己方位置、HP、状态和毁伤结果一致。
- [ ] 红蓝客户端抓取 WebSocket 数据时不存在未探测敌方的单位描述、坐标或消息。
- [ ] 敌方进入侦察范围后出现在对应阵营客户端，后续可见性符合既有共享侦察规则。
- [ ] 红蓝只能控制己方存活单位，猜测敌方 ID、越界坐标和不可见目标均被拒绝。
- [ ] 导演可暂停、继续、设置 0/1/2/4/8 倍速并单步推进暂停中的推演。
- [ ] 指挥所被摧毁后胜负事件只出现一次，导演仍可执行“结束并重置”。
- [ ] 导演结束后时间回到 0，双方单位、HP、路径和位置恢复到本轮开局快照。

## 实时通信与 UI

- [ ] 四类联网席位可互相发送中文、英文和混合文本，消息顺序和发送者席位正确。
- [ ] 空消息不会发送，超过 500 字被限制，快速刷屏触发服务端限流提示。
- [ ] 新登录用户能看到最近聊天记录，容器重启后聊天历史按设计清空。
- [ ] 1100x720、1360x860 和高 DPI 窗口下登录、顶部栏、阵容编辑、导演控制和聊天不重叠。
- [ ] 红蓝准备阶段地图遮罩、就绪按钮、阵容编辑对话框和推演阶段切换状态一致。
- [ ] 管理网页在常见桌面宽度下表格、弹窗、密码页无文字溢出或控件遮挡。
- [ ] 管理员可在“服务器监控”查看兵棋服务状态、连接日志和消息流；普通兵棋账号无此接口权限。
- [ ] 容器运维终端需要再次验证管理员密码；验证后可在 `account-web` 容器的非特权 `wargame` 用户 Shell 中执行命令，不能访问宿主机、Docker 套接字或 `game-server` 容器。
